# Личный кабинет франчайзера — PySide6

Десктоп-приложение для управления сетью торговых автоматов.  
Стек: **Python 3.13 · PySide6 · PostgreSQL · psycopg2 · openpyxl · bcrypt · PyJWT · python-dotenv**

---

## Как запустить

```bash
python main.py
```

Перед запуском нужен файл `.env` в корне проекта:

```
DB_HOST=localhost
DB_PORT=5432
DB_NAME=pyside6practice
DB_USER=postgres
DB_PASSWORD=root
APP_SECRET_KEY=любой_секретный_ключ
APP_JWT_EXPIRE_MINUTES=60
```

## Архитектура: слои

```
[ UI — app/ ]  →  [ Services — services/ ]  →  [ Core — core/ ]  →  [ PostgreSQL ]
```

| слой | что делает |
|------|-----------|
| **UI** | виджеты Qt, страницы, диалоги. только отображает данные и реагирует на действия пользователя |
| **Services** | бизнес-логика: SQL-запросы, подготовка данных. UI вызывает сервис, сервис возвращает dict/list |
| **Core** | инфраструктура: соединение с БД, логирование, настройки, безопасность |

**правило**: ui никогда не пишет sql напрямую. services не знают про Qt.

## Как работает авторизация

1. пользователь вводит email + пароль в `LoginDialog`
2. вызывается `authenticate(email, password)` из `auth_service.py`
3. сервис ищет пользователя в бд, проверяет пароль (поле `password_plain`)
4. если всё ок — создаётся JWT-токен (`create_access_token` в `security.py`)
5. данные пользователя кладутся в глобальный объект `session` (`session.py`)
6. `LoginDialog.accept()` закрывает диалог с кодом «успех»
7. `app/main.py` видит успешный ответ и открывает `MainWindow`

---

## Как работает навигация

1. `Sidebar` — виджет с кнопками разделов. при нажатии кнопки эмитит сигнал `menu_selected(key)`
2. `MainWindow` подключает слот `_on_menu` к этому сигналу
3. `_on_menu` переключает `QStackedWidget` на нужную страницу и обновляет хлебную крошку
4. раздел «администрирование» виден только роли `admin` — `set_admin_visible(is_admin)`

## Роли пользователей

| роль | что может |
|------|-----------|
| `admin` | всё: включая раздел «администрирование» |
| `operator` | просмотр + экспорт Excel, без администрирования |
| `viewer` | только просмотр, кнопки экспорта скрыты |

Проверка роли происходит в двух местах:
- `main_window.py` — скрывает кнопку «администрирование» в меню
- страницы `monitor_page.py`, `reports_page.py`, `inventory_page.py` — скрывают кнопку «скачать excel»

## Главная страница (Dashboard)

5 плиток, каждая читает данные из своей функции сервиса:

| плитка | сервис | что внутри |
|--------|--------|-----------|
| эффективность сети | `network_efficiency()` | процент работающих ТА из `vw_network_efficiency` |
| состояние сети | `network_status_breakdown()` | круговая диаграмма по статусам из `vw_status_breakdown` |
| сводка | `summary_cards()` | три цифры из `vw_summary` |
| динамика продаж | `sales_last_10_days(metric)` | столбчатый график из `vw_sales_last_10_days_*` |
| новости | `franchise_news(limit)` | последние записи из таблицы `news` |

плитки можно скрывать чекбоксами и перетаскивать мышью (QListWidget с DragDrop).

## Экспорт данных

реализован в трёх страницах:

| страница | форматы |
|----------|---------|
| монитор та | xlsx |
| детальные отчёты | xlsx |
| учёт тмц | xlsx |
| **список та (admin)** | **xlsx, csv, html, pdf** |

для xlsx используется библиотека `openpyxl` (импортируется только при экспорте).  
pdf генерируется через `QPdfWriter` + `QTextDocument` — рендерим html и «печатаем» в pdf.  
csv сохраняется в кодировке `utf-8-sig` (с BOM), чтобы Excel на Windows открывал без проблем с кириллицей.

## Пагинация (список ТА)

- `offset` — с какой записи начинать текущую страницу
- `limit` — сколько записей показывать (10/20/50, выбирается в комбобоксе)
- сервис `list_machines()` делает два запроса: `COUNT(*)` для `total`, потом `SELECT ... LIMIT ... OFFSET ...`
- кнопка `<` уменьшает offset на limit, `>` — увеличивает (если не вышли за total)

## Контекстный менеджер `db_cursor`

```python
# чтение (без commit)
with db_cursor() as cur:
    cur.execute('SELECT ...')
    rows = cur.fetchall()

# запись (с commit)
with db_cursor(commit=True) as cur:
    cur.execute('INSERT ...')
```

- `RealDictCursor` — строки возвращаются как `dict`, не как `tuple`
- если внутри блока ошибка — автоматический `rollback`, бд не ломается
- соединение закрывается в `finally` — всегда, даже при ошибке

## Таблицы БД (основные)

| таблица | описание |
|---------|---------|
| `users` | пользователи системы (id, email, role, password_plain, is_active) |
| `user_profiles` | фото профиля (photo_base64), привязан к users |
| `companies` | компании-франчайзи |
| `vending_machines` | торговые автоматы (модель, статус, адрес, даты, и т.д.) |
| `modems` | gsm/wifi-модемы, привязываются к автоматам |
| `products` | товары с ценой и минимальным запасом |
| `machine_stock` | остатки товаров в каждом автомате |
| `sales` | записи о продажах (товар, автомат, дата, сумма, способ оплаты) |
| `payment_methods` | справочник способов оплаты |
| `machine_monitor_snapshot` | оперативный снапшот состояния автомата (деньги, загрузка, события) |
| `news` | новостная лента для главной страницы |

**представления (views)** для dashboard:
- `vw_network_efficiency` — процент работающих та
- `vw_status_breakdown` — количество та по статусам
- `vw_summary` — сводные цифры
- `vw_sales_last_10_days_amount` / `vw_sales_last_10_days_quantity` — данные для графика

## Скрипт импорта данных (`db/import/import_data.py`)

одноразовый скрипт для первичного заполнения базы данных из файлов (csv/json/xlsx).  
запускается вручную один раз при развёртывании проекта.  
путь к файлам задаётся через переменную `IMPORT_ROOT` в `.env`.
