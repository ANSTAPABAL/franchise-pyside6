Проект: подготовка к ДЭ по КОД 09.02.07-2-2026 (Том 1)
Специальность: 09.02.07 Информационные системы и программирование

====================================================================
0) ЧТО ВЗЯТО ИЗ КРИТЕРИЕВ КОД (ВАЖНО)
====================================================================

По PDF (КОД 09.02.07-2-2026 Том 1) ключевые ориентиры:
- Базовый уровень ГИА: 3 часа.
- Профильный уровень (инвариант): 4 часа.
- Максимум ПУ с вариативной частью: 5 часов.
- Модули:
  1) Разработка БД средствами СУБД.
  2) Разработка алгоритма и создание приложения.
  3) Последовательный UI + CRUD товаров (для БУ/ПУ).
  4) Интеграция модулей / заказы (ПУ).
- Обязательные практические результаты:
  - исходный код (НЕ архив),
  - исполняемые файлы,
  - SQL-скрипт БД,
  - ER-диаграмма (PDF),
  - блок-схема алгоритма (PDF),
  - скриншоты работы (DOCX).

====================================================================
1) ПРИОРИТЕТЫ ПО БАЛЛАМ И РИСКАМ
====================================================================

1.1. Самый высокий приоритет (делать первым)
- БД в 3НФ + PK/FK + ссылочная целостность.
- Импорт данных из "import" файлов.
- Рабочая авторизация (логин/пароль из БД, роли).
- Список товаров из БД.
- Базовый функционал модуля 2: алгоритм + рабочее приложение.

1.2. Средний приоритет
- Поиск/фильтрация/сортировка в реальном времени.
- CRUD товаров (особенно add/edit/delete).
- Обработка ошибок и информативные сообщения.
- Единый стиль интерфейса и корректные заголовки окон.

1.3. Профильный/дополнительный приоритет
- Модуль заказов (роль менеджер/админ).
- Интеграционные сценарии между модулями.
- Дополнительные отчеты/экспорты/улучшения UX.

====================================================================
2) МАППИНГ MINDMAP -> КОД ДЭ
====================================================================

2.1 Python / OOP / PySide6
- Нужны для Модуля 2 и Модуля 3:
  - окна, виджеты, сигналы/слоты,
  - структура классов,
  - последовательная навигация между страницами.

2.2 psycopg2 / PostgreSQL / pgAdmin
- Нужны для Модуля 1:
  - создание схемы БД,
  - импорт данных,
  - SQL-запросы приложения,
  - получение ER-диаграммы и SQL-скриптов.

2.3 Linux / установка ПО
- Нужны как среда проведения:
  - заранее проверить окружение, версии ПО,
  - подготовить шаблоны команд и скрипты запуска.

2.4 Артефакты (из твоей mindmap "Приложение")
- роли пользователей,
- код приложения,
- SQL-скрипт БД,
- ER-диаграмма PDF,
- блок-схема PDF,
- скриншоты в DOCX.
Это полностью совпадает с тем, что реально спрашивают в КОД.

====================================================================
3) ЭТАПЫ ПОДГОТОВКИ К ДЭ (ПРАКТИЧЕСКИЙ ПЛАН)
====================================================================

Этап A. Шаблон проекта (must have)
1) Подготовить заготовку структуры:
   app/, services/, core/, db/sql/, db/import/, docs/.
2) Шаблон подключения к БД (settings + db_cursor).
3) Шаблон логина и сессии.

Этап B. Модуль 1: БД (самое важное)
1) Спроектировать ER-модель (согласованные имена).
2) Реализовать SQL: таблицы, ключи, ограничения.
3) Импортировать исходные файлы.
4) Сгенерировать ER в PDF.
5) Проверить SQL-скрипт на "чистый запуск" с нуля.

Этап C. Модуль 2: Алгоритм + каркас приложения
1) Нарисовать блок-схему (ГОСТ 19.701-90) в PDF.
2) Реализовать окно входа + роли.
3) Реализовать основной shell (навигация, заголовки, ФИО справа сверху).
4) Подключить список товаров из БД.

Этап D. Модуль 3: UI-функционал
1) Поиск, фильтрация, сортировка (в реальном времени).
2) CRUD товара (add/edit/delete), ограничения полей, проверки.
3) Обновление списка после CRUD.
4) Сообщения об ошибках/предупреждениях/инфо.

Этап E. Модуль 4 (ПУ)
1) Экран заказов (менеджер/админ).
2) CRUD заказов.
3) Ограничения доступа по ролям.

Этап F. Упаковка результатов
1) Снимки экранов в DOCX.
2) Проверка исполняемого файла.
3) Проверка структуры репозитория (не архив, а папки/файлы).

====================================================================
4) ЧТО ТРЕНИРОВАТЬ ПОД ТАЙМИНГ ЭКЗАМЕНА
====================================================================

Тренировка #1 (1.5 часа, ПА):
- только модуль БД + базовый UI.

Тренировка #2 (3 часа, ГИА БУ):
- БД + модуль 2 + часть модуля 3 (поиск/фильтр/сортировка).

Тренировка #3 (4 часа, ГИА ПУ):
- БД + модуль 2 + модуль 3 + основа модуля 4.

На каждой тренировке:
- 15 минут в конце на артефакты (PDF/DOCX/проверка запуска).

====================================================================
5) ЧТО КРИТИЧНО, ЧТО ОПЦИОНАЛЬНО
====================================================================

Критично (без этого теряешь много баллов):
- рабочая БД (3НФ, ключи, импорт),
- авторизация и роли,
- список данных из БД,
- поиск/фильтр/сортировка,
- базовый CRUD,
- корректная обработка ошибок,
- обязательные документы (ER PDF, блок-схема PDF, DOCX скриншоты).

Опционально (улучшает, но не должно съедать время):
- сложный дизайн-анимations,
- лишние фичи, не требуемые модулем,
- глубокий рефакторинг посреди экзамена.

====================================================================
6) ДОРОЖНАЯ КАРТА ОБУЧЕНИЯ (ЧТО УЧИТЬ)
====================================================================

Блок 1 (обязательный минимум):
1) SQL: CREATE TABLE, PK/FK, JOIN, GROUP BY, VIEW, INSERT/UPDATE/DELETE.
2) PySide6: окна, формы, таблицы, сигналы/слоты, переходы между страницами.
3) Python: функции, классы, обработка ошибок.
4) Архитектура: UI -> service -> DB.

Блок 2 (для уверенного результата):
5) Валидация ввода, UX-сообщения, сценарии ошибок.
6) Импорт данных и преобразование форматов.
7) Ролевой доступ (кто что видит и может делать).
8) Быстрая генерация артефактов (ER PDF, блок-схема PDF, DOCX).

Блок 3 (для профиля и запаса):
9) Интеграция модулей и модуль заказов.
10) Git-процесс и аккуратная сдача результата в репозиторий.

====================================================================
7) ЧЕКЛИСТ ПЕРЕД ДЕМОЭКЗАМЕНОМ (ДЕНЬ -1)
====================================================================

Технический:
- проект запускается с нуля на "чистой" БД;
- SQL-скрипт разворачивает схему без ручных правок;
- импорт данных работает;
- логин/роли работают;
- нет критических падений при CRUD.

Документы:
- ER-диаграмма PDF готова;
- блок-схема PDF готова;
- DOCX со скриншотами готов;
- структура артефактов понятна и разложена по папкам.

Организационный:
- знаешь, что делать в первые 30 минут;
- знаешь, какие части можно отложить, если не хватает времени.

8) ПЕРВЫЕ 30 МИНУТ НА САМОМ ЭКЗАМЕНЕ
====================================================================

1) Прочитать задание и сразу пометить: must-have / optional.
2) Создать/проверить БД-скелет.
3) Поднять каркас приложения + login.
4) Вывести базовый список товаров из БД.
5) После этого идти по модулям по приоритету.

Главный принцип:
Сначала закрыть обязательные критерии по баллам, потом "красоту" и расширения.
